DESIGN GOALS

Currently, missionaries face logistical difficulties when connecting with their supporters. To manage their support, they must juggle a variety of different platforms, not only for providing updates to supporters (e-mail, social media, phone calls, etc.) but also for handling donations (cash, check, online payment platforms, etc.). Klema.io seeks to streamline the process by providing all of these functionalities in the same place.

SCREENS

\ HOME \

The activity feed serves as the home page for klema.io. Supports can see new blog updates by missionaries they choose to follow and support. Prayer requests are also displayed along the right side, sorted by recency.

\ PROFILE \

Each user has a personal profile as well, which includes a photo and personal information (a short biography, experience, interests, etc.). From this page, users can view recent prayer requests as well as recent posts, helping people find all they need to know about a missionary they support.

\ BLOGS \

One of klema.io's main functionalities is the ability to write and read blog posts, which can also include pictures. Users can view and like blog posts written by others. This helps missionaries easily provide updates for their supporters.

\ COMMUNITY \

Klema helps people of the same church, organization, or missions team find each other and stay connected through communities. The community page provides a description of the community, and also displays posts and prayer requests by its members.

\ NOTIFICATIONS \

Users have their own personal settings which can be accessed via the navbar dropdown. Interactive notifications help notify the user of any changes that have been made to their settings, or errors that occur behind the scenes.
